Phase 2
Upload Your Site to GitHub
~10 minutes · Free
▶
ℹ️ GitHub is where your website files live. Vercel (Phase 3) reads from GitHub to publish your site. Think of GitHub as your "source" and Vercel as the "publisher". Both are free.
🐙
Create a GitHub account Free
Go to github.com and sign up if you don't have an account. Use your [email protected] email (once it's set up) or your personal email for now.
🔗 github.com → Sign Up
📁
Create a new repository called "predicare-website"
Click the green "New" button on your GitHub dashboard. Name it exactly:
predicare-website
Set it to Public. Check "Add a README file". Click "Create repository".
⬆️
Upload your landing page file Critical
In your new repository, click "Add file" → "Upload files". Upload the file you downloaded from this kit. Rename it to exactly:
index.html
This name is critical — Vercel looks for index.html as the homepage automatically. Then click "Commit changes".
💡 Also upload the MVP prototype file as assess.html — this will make it available at predicare.in/assess automatically.

✅ Phase 2 Complete when:

You can see index.html (and assess.html) inside your predicare-website repository on github.com. Move to Phase 3.

Phase 3
Deploy on Vercel — Your Site Goes Live
~15 minutes · Free
▶
▲
Go to vercel.com and sign up with GitHub Free
Click "Sign Up" → "Continue with GitHub". This connects Vercel to your GitHub account automatically. The free tier is generous — more than enough for your launch.
🔗 vercel.com → Sign Up with GitHub
🔗
Import your predicare-website repository
On your Vercel dashboard, click "Add New Project". You'll see your GitHub repositories listed. Click "Import" next to predicare-website.
ℹ️ On the configuration screen, leave all settings as default. Framework Preset: "Other". Root Directory: "./". Click Deploy.
⚡
Click Deploy — wait 60 seconds
Vercel will build and deploy your site in about 60 seconds. You'll see a success screen with a temporary URL like:
predicare-website-abc123.vercel.app
Click that URL. Your website is now live on the internet. Share it with 5 people immediately while you continue with Phase 4.
💡 Every time you update index.html on GitHub, Vercel automatically redeploys within 30 seconds. This is your publish workflow going forward — edit file → push to GitHub → live in 30 seconds.

✅ Phase 3 Complete when:

You can open predicare-website-XXXX.vercel.app in a browser and see your PrediCare landing page. Move to Phase 4 to connect your real domain.

Phase 4
Connect predicare.in to Your Live Site
~10 minutes · Free
▶
ℹ️ Right now your site is at a Vercel URL. This phase connects predicare.in so your site appears at the real domain. DNS changes can take 5 minutes to 48 hours to propagate — usually under 30 minutes in India.
⚙️
Add your domain in Vercel settings
In your Vercel project, go to Settings → Domains. Type in:
predicare.in
Also add:
www.predicare.in
Vercel will show you DNS records you need to add. Keep this tab open.
🔧
Add DNS records in BigRock / GoDaddy
Log into your domain registrar. Go to DNS Management for predicare.in. Add these records exactly as shown by Vercel:
A Record (for predicare.in)
Type: A
Host: @ (or leave blank)
Value: 76.76.21.21 (Vercel's IP — confirm in your Vercel dashboard)
TTL: 600
CNAME Record (for www.predicare.in)
Type: CNAME
Host: www
Value: cname.vercel-dns.com
TTL: 600
⚠️ Always use the exact values Vercel shows you — the IP address above is an example. Vercel's actual values shown in your dashboard are what matter.
⏳
Wait for DNS propagation (5–60 minutes)
Go back to Vercel. The domain status will change from "Invalid" to a green checkmark when it's connected. You can also check propagation status at:
🔗 whatsmydns.net → enter predicare.in → check A record
When you see green checkmarks across India/Asia, your site is live at predicare.in. Vercel automatically adds free HTTPS/SSL — no action needed.

✅ Phase 4 Complete when:

You can open predicare.in in a browser and see your website with the padlock (🔒) icon in the address bar. If it shows "Not Secure" — wait another 10 minutes for SSL to provision.

Add Google Analytics to track visitors
Go to analytics.google.com → Create Property → Web → predicare.in. Copy your Measurement ID (looks like G-XXXXXXXXXX). Add this script before the </head> tag in your index.html:
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXXX');
</script>
Now you'll see live visitor data — where they came from, how long they stayed, which city. Essential for understanding your early traction.
